# 陈叔叔工具箱 - 增量更新功能说明

## 功能概述

工具箱支持通过 GitHub Release 进行增量更新，用户无需下载完整安装包，只需下载变化的文件即可完成更新。

## 工作原理

```
用户点击"检查更新"
        ↓
程序调用 GitHub API 获取最新 Release
        ↓
下载 manifest.json（文件清单）
        ↓
对比本地文件 SHA256
        ↓
只下载有变化的文件
        ↓
替换本地文件，完成更新
```

## 发布新版本步骤

### 1. 准备更新文件

如果本次更新涉及新增工具，将新工具文件放在 `patches/` 目录：

```
patches/
└── 新增工具/
    └── 某个工具.exe
```

### 2. 生成 manifest.json

```bash
# 只更新主程序（小版本更新）
python 发布脚本/generate-manifest.py 1.1.0

# 更新主程序 + 新增工具
python 发布脚本/generate-manifest.py 1.1.0 --patches ./patches

# 完整更新（包含所有工具，不推荐）
python 发布脚本/generate-manifest.py 1.1.0 --include-tools
```

这会生成 `manifest.json` 文件，包含所有文件的 SHA256 哈希值。

### 3. 编辑 manifest.json

打开生成的 `manifest.json`，补充以下信息：

```json
{
  "version": "1.1.0",
  "releaseDate": "2026-06-30",
  "changelog": "1. 修复了xxx问题\n2. 新增了xxx功能",
  "minUpgradeVersion": "1.0.0",
  "files": { ... }
}
```

- `changelog`: 更新日志（显示给用户）
- `minUpgradeVersion`: 最低可升级版本（低于此版本需要完整安装）

### 4. 创建 GitHub Release

1. 在 GitHub 上创建新的 Release
2. Tag 选择 `v1.1.0` 格式
3. 上传以下文件：
   - `manifest.json`（必须）
   - `陈叔叔工具箱-完整安装包.exe`（可选，给需要完整安装的用户）
   - `patches/` 目录下的所有文件

### 5. Release 文件结构示例

```
Release v1.1.0
├── 陈叔叔工具箱-完整安装包.exe    (完整安装，可选)
├── manifest.json                  (增量更新清单，必须)
└── patches/                       (变更文件)
    ├── 陈叔叔工具箱.exe
    └── 其他变化的文件...
```

## manifest.json 格式说明

```json
{
  "version": "1.1.0",           // 版本号
  "releaseDate": "2026-06-30",  // 发布日期
  "changelog": "更新说明",       // 更新日志
  "minUpgradeVersion": "1.0.0", // 最低可升级版本
  "files": {
    "陈叔叔工具箱.exe": {
      "sha256": "abc123...",     // 文件 SHA256 哈希值
      "size": 142685064,         // 文件大小（字节）
      "url": "patches/陈叔叔工具箱.exe"  // 下载路径（相对于 Release）
    },
    "logo.ico": {
      "sha256": "def456...",
      "size": 1016964,
      "url": "patches/logo.ico"
    }
  }
}
```

## 配置 GitHub 仓库

在代码中需要配置 GitHub 仓库信息：

1. 打开 `Controls/SettingsPanel.xaml.cs`
2. 找到 `UpdateChecker` 的构造函数调用
3. 替换为实际的 GitHub 用户名和仓库名：

```csharp
_checker = new UpdateChecker("你的GitHub用户名", "陈叔叔工具箱");
```

同样修改 `MainWindow.xaml.cs` 中的 `CheckForUpdatesOnStartup` 方法。

## 注意事项

1. **SHA256 校验**：程序会自动校验下载文件的 SHA256，确保文件完整性
2. **备份机制**：更新前建议备份重要文件
3. **重启更新**：主程序更新后需要重启才能生效
4. **网络问题**：如果 GitHub 访问困难，用户可以选择下载完整安装包

## 常见问题

### Q: 为什么检查更新失败？
A: 可能的原因：
- 网络连接问题
- GitHub API 速率限制（未认证 60次/小时）
- 仓库未公开

### Q: 如何跳过某个版本？
A: 程序会自动对比当前版本和最新版本，如果用户跳过多个版本，程序会下载所有中间版本的变更文件。

### Q: 更新失败怎么办？
A: 可以下载完整安装包手动安装，或者删除 `settings.json` 重置设置。
